Design challenge

The client has a logo, an .SVG image of the railway system, and a list of requirements. The goal is to prototype a design for a mobile travel planner app that matches as many requirements as possible and utilises the provided assets.

Prototyping

  • Rapidly designed a rough Figma prototype to test core layout and navigation.
  • Refined the lo-fi prototype into a hi-fi version with user feedback in mind.
  • Implemented real content, consistent theming, map interactivity and more refined visuals.

User testing and heuristic evaluation

  • Created structured testing protocols with tasks, interview questions, and debriefs.
  • Conducted two testing rounds to identify usability issues via recorded user interviews and issue severity ratings.
  • Addressed layout and navigation issues, implemented missing features (dropdown buttons, narrow paddings, trip cancellation feature, etc.)

Design thinking

  • Centred app flow around the interactive map for intuitive navigation.
  • Balanced visual clarity with constraints set by design requirements (e.g. colour choices inspired by source SVG).
  • Designed a minimal overlay-based layout to reduce clutter and improve navigation.
